Utilities

Single-purpose helper classes for display, spacing, typography, color, and more. All classes use the wb- prefix and most use !important to safely override component styles.

Display Flex Width & Height Overflow Margin Padding Typography Color Border & Radius Shadow Position Cursor Opacity Accessibility Responsive Visibility

Display

Control the CSS display property of any element.

wb-block wb-inline wb-inline-block wb-flex ▸ wb-inline-flex 
<div class="wb-hidden"></div>       <!-- display: none -->
<div class="wb-block"></div>       <!-- display: block -->
<span class="wb-inline"></span>    <!-- display: inline -->
<span class="wb-inline-block"></span> <!-- display: inline-block -->
<div class="wb-flex"></div>        <!-- display: flex -->
<span class="wb-inline-flex"></span> <!-- display: inline-flex -->
<div class="wb-grid-display"></div> <!-- display: grid -->

All display utilities use !important.

Width & Height

Sizing shortcuts for common layout needs.

wb-w-full — 100%
wb-w-auto shrinks to content 
<div class="wb-w-full"></div>   <!-- width: 100% -->
<div class="wb-w-auto"></div>   <!-- width: auto -->
<div class="wb-h-full"></div>   <!-- height: 100% -->
<div class="wb-min-w-0"></div>  <!-- min-width: 0  (fixes flex truncation) -->
Tip: wb-min-w-0 is essential inside flex containers when the child contains truncated text — without it, flex items won't shrink below their content size.

Position

CSS positioning and inset-0 shorthand for full-coverage overlays.

<div class="wb-relative"></div>  <!-- position: relative -->
<div class="wb-absolute"></div>  <!-- position: absolute -->
<div class="wb-fixed"></div>     <!-- position: fixed -->
<div class="wb-sticky"></div>    <!-- position: sticky -->
<div class="wb-inset-0"></div>   <!-- inset: 0 (top/right/bottom/left: 0) -->
Common pattern: combine wb-absolute wb-inset-0 to stretch an element to fill its nearest positioned ancestor — useful for overlays and full-cover backgrounds.

Accessibility

Visually hide content while keeping it available to screen readers.

<span class="wb-sr-only">Close menu</span>
wb-sr-only uses a standard visually-hidden technique: the element is positioned off-screen (1×1px, clipped, no overflow). Screen readers announce it; sighted users don't see it. Use it for icon-only buttons, decorative separators, and label supplements.

Responsive Visibility

Show or hide elements at specific breakpoints.

Resize your browser window to see these classes take effect.
wb-hidden-sm (hidden on mobile ≤600px) wb-hidden-lg (hidden on desktop ≥901px) wb-visible-mobile (visible on mobile only) wb-visible-desktop (visible on desktop only) 
<!-- Hide at breakpoint -->
<div class="wb-hidden-sm">Hidden on mobile (≤600px)</div>
<div class="wb-hidden-md">Hidden on tablet (601–900px)</div>
<div class="wb-hidden-lg">Hidden on desktop (≥901px)</div>

<!-- Show only at breakpoint -->
<div class="wb-visible-mobile">Visible on mobile only (≤768px)</div>
<div class="wb-visible-desktop">Visible on desktop only (>768px)</div>

Breakpoints: sm ≤ 600px  ·  md 601–900px  ·  lg ≥ 901px
