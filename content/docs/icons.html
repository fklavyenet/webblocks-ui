Two usage paths

Icons can stay semantic or quick.

WebBlocks ships icons both as an SVG sprite and as CSS classes for <i> tags. Choose the pattern that matches the page: semantic SVG when structure matters, class-based icons when speed matters.

Formats

  • Sprite

    Use <svg><use> for semantic, accessible icon markup.

  • Class based

    Use <i class="wb-icon wb-icon-search"> for faster repeated markup.

  • Single source

    Both forms come from the same generated icon source.

Examples

Both patterns stay explicit in the HTML.

Nothing here depends on an icon font or a hidden component wrapper.

<svg class="wb-icon wb-icon-sm" aria-hidden="true">
  <use href="#wb-icon-search"></use>
</svg>
<i class="wb-icon wb-icon-search wb-icon-sm" aria-hidden="true"></i>

Action Icons

Keep refresh, sync, and rotation separate in meaning.

Use refresh when the page fetches current data again, sync when the UI reconciles with a remote source, and rotate-cw when the action is literally cyclical or rotational. Retry can reuse refresh in the UI, but the action label should still say retry.

refresh
sync
rotate-cw
<button class="wb-btn wb-btn-secondary wb-icon-btn">
  <i class="wb-icon wb-icon-refresh wb-icon-sm"></i>
  Refresh data
</button>

<button class="wb-btn wb-btn-outline wb-icon-btn">
  <i class="wb-icon wb-icon-sync wb-icon-sm"></i>
  Sync remote
</button>

<button class="wb-btn wb-btn-ghost wb-icon-btn">
  <i class="wb-icon wb-icon-refresh wb-icon-sm"></i>
  Retry job
</button>

<button class="wb-btn wb-btn-ghost wb-icon-btn">
  <i class="wb-icon wb-icon-rotate-cw wb-icon-sm"></i>
  Rotate key
</button>
