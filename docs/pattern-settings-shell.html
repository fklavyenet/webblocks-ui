Settings shell

Use the settings shell when the page needs slower form work, grouped decisions, and a local section rail that stays close to the active task.

Why this shell exists Settings pages usually need a calmer navigation rail, clearer section grouping, and enough width for forms without turning into a full application dashboard.

Choose this shell for profile settings, billing preferences, notification rules, workspace options, and other pages where users move through grouped controls instead of hopping across dense application views.

Local workspace rail

The local rail is a real wb-section-nav, not a custom settings-only component. It keeps nearby settings areas visible without competing with the global docs or app navigation.

Good fit
  • Grouped form sections with short descriptions
  • Preferences that benefit from local navigation
  • Destructive actions isolated at the end of the page
