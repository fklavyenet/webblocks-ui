Ajustes finales

Las utilidades sirven para correcciones pequenas, no para un segundo sistema de diseno.

WebBlocks incluye helpers para display, flex, spacing, typography, color, border, shadow, position y visibility. Su trabajo es cerrar una composicion, no reemplazar la capa de componentes ni la de layouts.

Buen uso

  • Primero estructura

    Usa primero los primitives de layout y los defaults de componentes, luego las utilidades.

  • Mantenlas locales

    Una utility encaja bien cuando una o dos correcciones resuelven la pagina.

  • No apiles a ciegas

    Si aparecen demasiadas utilidades juntas, la decision probablemente pertenece a otra capa.

Grupos

Los helpers cubren los ajustes finales mas comunes.

La nomenclatura sigue el mismo ritmo del resto del sistema: corta, literal y con prefijo `wb-`.

  • display
  • flex
  • gap
  • margin
  • padding
  • text
  • font
  • color
  • border
  • radius
  • shadow
  • position
  • cursor
  • opacity
  • responsive visibility
Ejemplo

Unas pocas utilidades pueden cerrar un layout sin ocultarlo.

El uso sano es este: el componente hace casi todo el trabajo y las utilidades solo ajustan spacing y alignment.

<div class="wb-card wb-mt-4">
  <div class="wb-card-header wb-flex wb-items-center wb-justify-between wb-gap-3">
    <strong class="wb-text-lg">Actividad</strong>
    <span class="wb-badge wb-badge-success wb-badge-dot">En vivo</span>
  </div>
  <div class="wb-card-body wb-p-4">...</div>
</div>
Limite

Si el markup se convierte en una pared de helpers, la decision probablemente debe volver a la capa de componentes o layouts. El sistema es mas legible cuando las utilidades se usan con medida.
