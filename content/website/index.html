Source-visible by design

A UI system for server-rendered products that keeps the source visible.

WebBlocks is a source-visible UI system for server-rendered products: plain HTML, token-based CSS, and small vanilla JS modules for admin panels, internal tools, and structured product interfaces.

Read the docs Open examples

Best fit

Use WebBlocks when you want stable, inspectable markup instead of another abstraction layer.

  • For

    Laravel Blade, Twig, Django templates, Rails ERB, and plain HTML teams building admin and product surfaces.

  • With

    Named classes, token-based CSS, and small JS modules that remain readable in the browser.

  • Not

    A utility-first framework, a React/Vue abstraction library, or a marketing-page builder.

What it ships

Plain files, explicit structure, and a narrow toolset.

The project stays intentionally narrow: a token layer, layout primitives, UI primitives, patterns, an icon system, and small behavioral modules. The goal is not to invent another syntax.

Token-based CSS Foundation, theme axes, predictable styling
Color, spacing, type, radius, shadow, border, and theme choices stay visible and reusable.
Named primitives and shells Buttons, forms, tables, overlays, shells
The page structure remains explicit because the classes describe what the interface is doing.
Small vanilla JS modules Theme, modal, drawer, dropdown, toast, collapse
Behavior is opt-in, attribute-driven, and exposed through small public APIs.
